# Skills Vault Import Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Import the three user-defined Claude Code skills as `resource` nodes in the Superbrain vault, linked to the existing `Claude Setup` node, with a reusable sync script.

**Architecture:** A shell script (`scripts/import-skills.sh`) reads each skill directory from `~/.claude/skills/`, strips the YAML frontmatter, extracts the H1 title, and writes a vault note with the correct frontmatter into the vault's `resources/` folder. The `Claude Setup` note is then updated manually to link to each skill note. No app code changes needed.

**Tech Stack:** bash, awk, the existing Obsidian vault (iCloud path)

### Task 1: Write and run the import script

**Files:**
- Create: `scripts/import-skills.sh`

- [ ] **Step 1: Create the scripts directory and write the script**

```bash
mkdir -p scripts
```

Create `scripts/import-skills.sh` with this exact content:

```bash
#!/usr/bin/env bash
set -euo pipefail

SKILLS_DIR="$HOME/.claude/skills"
VAULT="$HOME/Library/Mobile Documents/iCloud~md~obsidian/Documents/Milan's Brain"
RESOURCES="$VAULT/resources"
TODAY=$(date +%Y-%m-%d)

for skill_dir in "$SKILLS_DIR"/*/; do
  [ -d "$skill_dir" ] || continue
  skill_file="$skill_dir/SKILL.md"
  [ -f "$skill_file" ] || continue

  dir_name=$(basename "$skill_dir")
  vault_note="$RESOURCES/skill-${dir_name}.md"

  # Extract H1 title (first line starting with "# "); fallback to dir name
  title=$(grep -m1 '^# ' "$skill_file" | sed 's/^# //' || echo "$dir_name")
  [ -z "$title" ] && title="$dir_name"

  # Strip YAML frontmatter: print lines after the second "---"
  # If no second "---" found (n never reaches 2), print the whole file
  body=$(awk '
    /^---/ { n++; next }
    n >= 2 { print }
  ' "$skill_file")
  if [ -z "$body" ]; then
    body=$(cat "$skill_file")
  fi

  cat > "$vault_note" <<EOF
---
title: ${title}
type: resource
tags: [tools, claude, skill]
date: ${TODAY}
relations:
  - target: "[[Claude Setup]]"
    type: part_of
---

${body}
EOF

  echo "✓ $vault_note"
done
```

- [ ] **Step 2: Make the script executable and create convenience symlink**

```bash

```

The symlink lets you run `import-to-vault.sh` directly from `~/.claude/skills/` without knowing the repo path.

## Running the sync in the future

When a skill is updated (either in the vault or locally), re-run:

```bash
./scripts/import-skills.sh
```

The script always overwrites. Do not run it while you have unsaved edits to the skill notes in the vault.
